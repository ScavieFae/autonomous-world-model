# EB-JEPA: Energy-Based Joint Embedding Predictive Architectures

**Paper:** arXiv 2602.03604 (February 2026)
**Authors:** Terver et al. (Facebook/Meta AI Research)
**Code:** https://github.com/facebookresearch/eb_jepa
**License:** CC-BY-NC 4.0

## Core Idea

Pedagogical library reframing JEPA through an energy-based model lens. Implements three worked examples progressing toward world models:
1. Image JEPA on CIFAR-10
2. Video JEPA on Moving MNIST
3. **Action-Conditioned Video JEPA** on Two Rooms environment (world modeling + planning)

## Architecture Variants

### Anti-Collapse: VC Regularization (instead of EMA)
Unlike V-JEPA (which uses EMA target encoder), EB-JEPA uses explicit Variance-Covariance regularization:
- **HingeStdLoss**: penalizes per-feature std below margin (prevents dimensional collapse)
- **CovarianceLoss**: penalizes off-diagonal covariance (prevents feature redundancy)
- **TemporalSimilarityLoss**: encourages smooth consecutive representations
- **InverseDynamicsLoss**: predicts actions from consecutive state pairs

### Predictor Options
- ResUNet for video (two consecutive frames → next frame representation)
- **RNNPredictor (GRU)**: single-step state propagation conditioned on actions — most relevant for world modeling

### Unrolling
Two modes in `JEPA.unroll()`:
- **Parallel**: all timesteps at once, ground truth re-fed from left (training)
- **Autoregressive**: step-by-step with sliding window, each step uses own predictions (inference/planning)

## Planning via CEM/MPPI in Latent Space

The action-conditioned example demonstrates model-predictive control:
1. Encode current observation and goal
2. Sample candidate action sequences
3. Unroll JEPA predictor for each sequence
4. Score by representation distance to goal embedding
5. Refine action distribution (CEM elite selection or MPPI reweighting)
6. Execute first action(s), repeat

## Relevance to AWM/Melee

- The AC Video JEPA example is a minimal working world model
- RNN predictor (GRU) conditioned on actions is analogous to our sequence model conditioned on controller inputs
- VC regularization is an alternative to SIGReg for preventing collapse
- The planning loop (CEM in latent space) maps to potential "The Wire" arena usage
- More transparent/modular than LeWM — good for understanding the building blocks
