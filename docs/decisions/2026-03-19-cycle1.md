# Decision Log: 2026-03-19 Cycle 1

3 hypotheses proposed and reviewed in parallel. 1 approved, 2 revised→approved with conditions.

## Experiment 1: e020b-sf-ratio-30

**Hypothesis:** 30% SF ratio (ratio:3, actual 25%) will improve RC below 6.03 by increasing error exposure frequency. E020a showed 10% is too little; 30% is the natural next test point.

**Director verdict:** APPROVED

**Reasoning:** Sound mechanism, distinct axis from prior SF failures (E018b tested unroll length, E018d tested loss weighting — neither tested ratio). Prior Director rejections of this axis (2026-03-15, 2026-03-16) were based on insufficient evidence. With E020a data showing 10% is below optimal, 30% is the principled next step.

**Concerns:** ratio:3 gives 25% not 30%. Watch gradient budget split (TF vs SF loss). If SF loss >0.5, training may be destabilized.

**Budget:** $5 Scout

## Experiment 2: e022a-bs256

**Hypothesis:** Halving batch size 512→256 (LR 5e-4→2.5e-4, linear scaling rule) will improve RC via implicit regularization and 2× gradient update frequency. Karpathy's single-largest improvement.

**Director verdict:** REVISE → APPROVED with conditions

**Conditions addressed:**
1. GPU locked to A100 40GB (~$5-6 cost)
2. LR=2.5e-4 documented as confound (linear scaling rule is an assumption)
3. RC prediction tightened to "improve below 6.00" (clearer falsification)

**Reasoning:** Untested axis, explicitly flagged in program.md §6. Grounded in external authority (Karpathy autoresearch). Core mechanism sound.

**Concerns:** LR scaling rule may not hold. More optimizer steps could cause overtraining. SF interaction uncharacterized.

**Budget:** $6 Scout

## Experiment 3: e023a-dmodel192

**Hypothesis:** Halving d_model 384→192 will maintain or improve RC at 1.9K data scale. Bias-variance tradeoff: smaller model may generalize better.

**Director verdict:** REVISE → APPROVED with conditions

**Conditions addressed:**
1. Parameter counts corrected (actual model is ~20M, not 4.3M — includes embeddings+heads)
2. Mechanism narrowed to SSM hidden state width capacity
3. Acknowledged E018c works at d_model=384 on 1.9K data

**Reasoning:** Phase 1 Tier 1 architecture grid, explicitly designed in program.md §7. Config-only, one-axis change. Will produce clear signal regardless of direction.

**Concerns:** Embedding bottleneck at narrow trunk. Convergence speed differences. Same LR may not be optimal.

**Budget:** $6 Scout

## What would change the verdict

- **All 3 rejected:** If we had evidence that all three axes were tested before (we don't — none have been tested).
- **e020b rejected:** If there were 2+ data points showing ratio changes regress. We have 1 (E020a at 10%), which regressed for the *opposite* reason (too little SF).
- **e022a rejected:** If batch size had been swept before. It hasn't.
- **e023a rejected:** If d_model had been tested before. It hasn't.
